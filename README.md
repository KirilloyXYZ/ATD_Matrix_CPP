# АТД Matrix C++

Лабораторная работа №3: полиморфный абстрактный тип данных для матриц на основе АТД последовательности из лабораторной работы №2.

## Возможности

- Абстрактный интерфейс `Matrix<T>`.
- Реализации `RectangularMatrix<T>` и `SquareMatrix<T>`.
- Операции над матрицами: сложение, умножение на скаляр, норма Фробениуса.
- Элементарные преобразования строк и столбцов.
- Решение систем линейных уравнений:
  - метод Гаусса без выбора ведущего элемента;
  - метод Гаусса с выбором ведущего элемента;
  - LU-разложение.
- Консольный интерфейс для создания матриц, выполнения операций, замера производительности и решения СЛАУ.
- Модульные тесты для последовательностей, битовых последовательностей, матриц и решателей СЛАУ.

## Сложность

Обозначения:

- `m` - количество строк матрицы;
- `n` - количество столбцов матрицы;
- для СЛАУ `n` - размер системы;
- цикломатическая сложность считается как `1 + количество точек ветвления` внутри тела функции.

### Операции над матрицами

| Операция | Временная сложность | Дополнительная память | Цикломатическая сложность |
| --- | --- | --- | --- |
| `Get`, `Set` | `O(1)` | `O(1)` | `1` |
| `Add` | `O(m * n)` | `O(m * n)` | `3` |
| `MultiplyByScalar` | `O(m * n)` | `O(m * n)` | `3` |
| `Norm` | `O(m * n)` | `O(1)` | `3` |
| `SwapRows` | `O(n)` | `O(n)` | `2` |
| `SwapColumns` | `O(m)` | `O(1)` | `3` |
| `MultiplyRow` | `O(n)` | `O(1)` | `2` |
| `MultiplyColumn` | `O(m)` | `O(1)` | `2` |
| `AddRowMultiple` | `O(n)` | `O(1)` | `2` |
| `AddColumnMultiple` | `O(m)` | `O(1)` | `2` |

### Решатели СЛАУ

| Функция | Временная сложность | Дополнительная память | Цикломатическая сложность |
| --- | --- | --- | --- |
| `MultiplyMatrixByVector` | `O(m * n)` | `O(m)` | `3` |
| `VectorNorm` | `O(n)` | `O(1)` | `2` |
| `ComputeResidualNorm` | `O(m * n)` | `O(m)` | `2` |
| `ComputeRelativeError` | `O(n)` | `O(n)` | `3` |
| `SolveGaussNoPivot` | `O(n^3)` | `O(n^2)` | `4` |
| `SolveGaussPartialPivot` | `O(n^3)` | `O(n^2)` | `7` |
| `DecomposeLU` | `O(n^3)` | `O(n^2)` | `6` |
| `ForwardSubstitution` | `O(n^2)` | `O(n)` | `3` |
| `BackwardSubstitution` | `O(n^2)` | `O(n)` | `3` |
| `SolveLU` | `O(n^3)` | `O(n^2)` | `1` |

`SolveLU` имеет цикломатическую сложность `1`, потому что это функция-обертка: ветвления и циклы находятся внутри `DecomposeLU`, `ForwardSubstitution` и `BackwardSubstitution`.
